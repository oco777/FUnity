# FUnity Modes

FUnity では制作者の目的に合わせて 2 種類の制作モードを提供しています。

- **ブロックモード (Block Mode)** — Scratch 風の 480x360 ピクセルステージでブロックプログラミングを行うモードです。SB3 インポートにも対応予定です。
- **unityroom モード** — 16:9 (推奨 960x540) のステージを用いて unityroom への WebGL 公開を最適化するモードです。Unity 2D 拡張機能を積極的に活用できます。

切り替えは `FUnityProjectData` アセットの Inspector で行ってください。
`FUnityProjectData` に登録された ModeConfig がゲーム起動時に自動適用されます。

## 各モードの用途と特徴

### ブロックモード（Scratch モード）

- 想定用途
  - CoderDojo などでの Scratch 風ブロックプログラミング学習
  - 1 画面内にステージとブロックエディタをまとめた教材的な使い方
- ステージサイズ / レイアウト
  - Scratch に近い 480x360 ピクセルのステージを基本とし、左右に余白を持つレイアウトを想定
- 座標系・角度
  - Scratch に近い座標系（中央原点）と角度の扱い（0=右, 90=上, 180=左, 270=下）を重視
  - ScratchAngleUtil による Scratch 角度 ↔ 内部角度の変換を利用
- UI
  - ステージ、変数モニタ、メッセージ、ブロック（Visual Scripting）を 1 画面にまとめた UI
- 想定ビルドターゲット
  - PC / タブレットなどでの教材用途・ローカル実行が主な想定

### unityroom モード

- 想定用途
  - unityroom への WebGL 公開を前提とした 1 本のゲーム制作
  - スコア・残機・タイマーなどを持つアクション／シューティングゲームなど
- ステージサイズ / レイアウト
  - 16:9 のステージ（推奨 960x540 など）を用いた一般的なゲーム画面レイアウト
- 座標系・角度
  - 基本的な 2D ゲームの座標系を意識しつつ、内部的には Scratch モードと共通の Presenter / View を利用
- UI
  - ゲーム画面を全面に出しつつ、スコアや残機などの HUD を自由に配置
  - ブロックエディタは別シーン／別ウィンドウで扱うことを想定
- 想定ビルドターゲット
  - WebGL ビルドで unityroom にアップロードする使い方を想定

## ブロックモードと unityroom モードの比較表

| 項目                        | ブロックモード（Scratch）                           | unityroom モード                                 |
|-----------------------------|------------------------------------------------------|--------------------------------------------------|
| 主な用途                    | 教材・学習用、Scratch 風ブロックプログラミング     | unityroom 公開用の本格的なゲーム                |
| ステージサイズ              | 480x360 ベースの固定ステージ                        | 16:9 ベースのゲーム画面                          |
| 座標系・角度の考え方        | Scratch の仕様に合わせた座標系・角度変換            | 一般的な 2D ゲーム寄り（内部処理は共通）        |
| UI レイアウト               | ステージ＋ブロックエディタ＋変数モニタ             | ゲーム画面＋HUD（ブロックエディタは別画面）     |
| 想定ビルドターゲット        | ローカル教材、デモ、学習用                          | WebGL（unityroom）                               |
| 初心者／子ども向け          | ◎（Scratch に近い操作感）                            | △〜○（ゲーム制作経験者向け〜中級以上）         |
| Scratch からの移行のしやすさ | ◎                                                 | ○（ブロックは流用可能だが画面レイアウト調整が必要） |

## どちらのモードを選ぶべきか

- 子ども向けワークショップや Scratch 体験会:
  - ブロックモード（Scratch モード）を推奨。
- unityroom のコンテストに参加したい、ブラウザ公開したい:
  - unityroom モードを推奨。
  - 最初はブロックモードで試作し、後から unityroom モード用プロジェクトに移植するワークフローも有効です。
